# Configuration Reference

Documentor uses a YAML configuration file, typically named `documentor.yaml`, to control how documentation is generated and managed. This file defines LLM settings, output directories, styling instructions, file tracking preferences, and the specific documentation files to be generated.

To create a configuration file interactively, run:

```bash
documentor init
```

## Example `documentor.yaml`

Full example of a `documentor.yaml` file:

```yaml
provider: vertexai
model: gemini-3.1-pro-preview
docs_dir: docs
include_footer: false
use_style_md: true
style_md_path: docs/style.md
use_git: true
ignore_above_size_kb: 100
ignore_patterns:
  - .git
  - __pycache__
  - venv
  - .venv
  - env
  - node_modules
  - .env
  - "*.pyc"
  - "*.pyo"
use_agent: false
agent_threshold_kb: 1000
required_files:
  - filename: API.md
    description: Core API reference for the application endpoints
    type: API Reference
  - filename: Overview.md
    description: High-level overview of the project architecture
    type: Architecture
```

## Configuration Options

Detailed reference of all available options in `documentor.yaml`.

### LLM Settings

Defines which AI model is used to generate documentation.

*   **`provider`** (string)
    *   **Description**: The LLM provider to use.
    *   **Allowed Values**: `openai`, `vertexai`, `ollama`
    *   **Default**: `vertexai`
*   **`model`** (string)
    *   **Description**: The specific model name to use from the chosen provider.
    *   **Default**: `gemini-3.1-pro-preview`

### Output & Styling

Configures where and how documentation is saved and styled.

*   **`docs_dir`** (string)
    *   **Description**: The directory where generated markdown documentation will be saved.
    *   **Default**: `docs`
*   **`include_footer`** (boolean)
    *   **Description**: Appends a "Generated by Documentor" footer to each generated markdown file.
    *   **Default**: `false`
*   **`use_style_md`** (boolean)
    *   **Description**: Reads a style guide markdown file and applies its formatting instructions when generating documents.
    *   **Default**: `false`
*   **`style_md_path`** (string)
    *   **Description**: The path to the style guide file. Only used if `use_style_md` is `true`.
    *   **Default**: `docs/style.md`

### Project & File Tracking

Controls how Documentor scans project source code.

*   **`use_git`** (boolean)
    *   **Description**: Enables Git-based tracking for incremental updates to detect changes since the last sync.
    *   **Default**: `true`
*   **`ignore_above_size_kb`** (integer)
    *   **Description**: Excludes source files larger than this size (in kilobytes) when scanning for context.
    *   **Default**: `100`
*   **`ignore_patterns`** (list of strings)
    *   **Description**: Standard file and directory patterns to ignore during context extraction.
    *   **Default**: `[".git", "__pycache__", "venv", ".venv", "env", "node_modules", ".env", "*.pyc", "*.pyo"]`

### Documentation Scope

Defines the documentation files Documentor generates and maintains.

*   **`required_files`** (list of objects)
    *   **Description**: The list of documentation files to create and keep updated. Run `documentor plan` to auto-generate this list based on the codebase.
    *   **Default**: `[]`
    *   **Object Structure**:
        *   **`filename`** (string): The target markdown filename (e.g., `Authentication.md`).
        *   **`description`** (string): A short description guiding the AI on what the document should cover.
        *   **`type`** (string): The category of the document (e.g., `API Reference`, `Tutorial`, `Architecture`).

### Agent Settings

Settings for handling very large projects using an iterative agent mode.

*   **`use_agent`** (boolean)
    *   **Description**: Manually forces the use of the agent-based system for context extraction instead of standard chaining.
    *   **Default**: `false`
*   **`agent_threshold_kb`** (integer)
    *   **Description**: Automatically switches to agent mode if the total size of source code context exceeds this threshold (in kilobytes) to prevent hitting LLM context limits.
    *   **Default**: `1000`
