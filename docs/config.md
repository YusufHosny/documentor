# Configuration Reference

Documentor uses a YAML configuration file, typically named `documentor.yaml`, to control how documentation is generated and managed. 

To create a configuration file interactively, run:

```bash
documentor init
```

## Example `documentor.yaml`

```yaml
provider: vertexai
model: gemini-3.1-pro-preview
docs_dir: docs
include_footer: false
use_style_md: false
style_md_path: docs/style.md
use_git: true
ignore_above_size_kb: 100
ignore_patterns:
  - .git
  - __pycache__
  - venv
  - .venv
  - env
  - node_modules
  - .env
  - "*.pyc"
  - "*.pyo"
use_agent: false
agent_threshold_kb: 1000
required_files:
  - filename: API.md
    description: Core API reference for the application endpoints
    type: API Reference
  - filename: Overview.md
    description: High-level overview of the project architecture
    type: Architecture
```

---

## Options

### LLM Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `provider` | string | `vertexai` | The LLM provider. Allowed values: `openai`, `vertexai`, `ollama`. |
| `model` | string | `gemini-3.1-pro-preview` | The specific model name. |

### Output & Styling

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `docs_dir` | string | `docs` | Directory where generated documentation is saved. |
| `include_footer` | boolean | `false` | Appends a "Generated by Documentor" footer to each markdown file. |
| `use_style_md` | boolean | `false` | Applies formatting instructions from a style guide file when generating documents. |
| `style_md_path` | string | `docs/style.md` | Path to the style guide file. Used only if `use_style_md` is `true`. |

### Project & File Tracking

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `use_git` | boolean | `true` | Enables Git-based tracking to detect incremental changes since the last sync. |
| `ignore_above_size_kb`| integer | `100` | Excludes source files larger than this size (KB) during context extraction. |
| `ignore_patterns` | list | `[".git", "__pycache__", "venv", ".venv", "env", "node_modules", ".env", "*.pyc", "*.pyo"]` | File and directory patterns to ignore during context extraction. |

### Documentation Scope

*   **`required_files`** (list of objects): The list of documentation files to keep updated. Run `documentor plan` to auto-generate this list based on the codebase. Default: `[]`.
    *   **`filename`** (string): The target markdown filename (e.g., `Authentication.md`).
    *   **`description`** (string): A brief description guiding the AI on what the document should cover.
    *   **`type`** (string): The document category (e.g., `API Reference`, `Architecture`).

### Agent Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `use_agent` | boolean | `false` | Forces the use of the iterative agent mode instead of standard chaining. |
| `agent_threshold_kb` | integer | `1000` | Automatically switches to agent mode if the source context size exceeds this threshold (KB) to prevent hitting LLM context limits. |

> *Docs generated with [documentor](https://github.com/YusufHosny/documentor)*
